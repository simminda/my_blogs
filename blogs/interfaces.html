Java Interfaces

October 27, 2021


What is an interface in Java and why would you, as a software developer, use interfaces?

An interface is like a blueprint of a class. It specifies a set of methods that the class has to implement. An interface can have variables and methods, just like a class, but the methods declared in an interface are by default abstract.

  • Interfaces are used to:
  • Implement abstraction. Used to achieve total abstraction.
  • Achieve loose coupling
  • Achieve multiple inheritance (Java Classes do not support multiple inheritance)

 

What is the difference between an abstract class and an interface?

Abstraction is hiding the internal implementation of a feature and only showing its functionality to users. Abstract class and Interface are both used for abstraction. Interfaces are used to achieve full abstraction whereas abstract classes achieve partial abstraction.

  • Interfaces can have only abstract methods. Interface variables can only be static and final. Interface members are public by default.
  • An abstract class can have abstract and non-abstract methods. Abstract class variables can be declared as non-final. Abstract classes can have final, non-final, static and non-static variables.
  • An interface can extend another interface only. An abstract class can extend another class and implement multiple interfaces.

 

Why is abstraction an important concept in software development and what role do interfaces play in abstraction?

Abstraction is one of the key concepts of object-oriented programming (OOP) languages. Its main goal is to handle complexity by hiding unnecessary details from the user. That enables the user to implement more complex logic on top of the provided abstraction without understanding or even thinking about all the hidden complexity (Jansen T , 2017). The appliances we use in our everyday lives all serve as an example of abstraction because we use them without knowing, or having to know, their internal workings and all the calculations and parameters they have to check in the background. Interfaces are used to achieve total abstraction. The methods declared in an interface are by default abstract and interfaces simulate multiple inheritance so they are very convenient when it comes to abstraction

 

What must a class do in order to implement an interface?

To declare a class that implements an interface, the class must provide all of the methods in the interface and identify the interface in its implements clause.

 

What is an abstract method?

As per docs.oracle.com:

An abstract method is a method that is declared without an implementation (without braces, and followed by a semicolon), like this:

abstract void moveTo(double deltaX, double deltaY);

If a class includes abstract methods, then the class itself must be declared abstract, as in:

public abstract class GraphicObject {

            // declare fields

            // declare nonabstract methods

            abstract void draw();

}

 

Can you instantiate an interface?

Interfaces cannot be instantiated. They are implemented.

Interfaces have no method implemented; therefore, there is no purpose in instantiating it as nothing would happen when a method is invoked

 

Can you declare a constructor inside an interface? If not, why?

The purpose of a constructor is to initialize the non-static members of a particular class with respect to an object. An interface cannot have a constructor because all the data members in interfaces are public static final by default.

 

Can we override an interface method with visibility that is not public?

We cannot override an interface method if its visibility is not public. If it has its visibility as public then you can override it with the same method signature (i.e. with the same access specifier public) whenever you implement the interface to any class.
